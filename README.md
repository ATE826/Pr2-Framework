## Суть задания (оценка 4)

Необходимо реализовать микросервисную серверную систему, состоящую из трёх компонентов: **API-шлюза**, **сервиса пользователей** и **сервиса заказов**. В рамках оценки 4 студент должен обеспечить полный функционал работы с пользователями и заказами, включая аутентификацию, профили и жизненный цикл заказов.

### Требуемый функционал для оценки 4

**1. Пользователи**

* Регистрация с валидацией входных данных.
* Вход и выдача JWT-токена.
* Доступ к защищённым маршрутам только при наличии токена.
* Получение и обновление собственного профиля.
* Админ-функция: список пользователей с фильтрами и пагинацией.

**2. Заказы**

* Создание заказа авторизованным пользователем.
* Получение заказа по ID (только своего).
* Получение списка своих заказов с пагинацией и сортировкой.
* Обновление статуса и отмена заказа (с проверкой прав).

### Инфраструктура и требования

* API-шлюз прокидывает запросы в сервисы, проверяет JWT, поддерживает CORS и X-Request-ID.
* Все ответы — в едином JSON-формате.
* Логи и трассировки включены во всех сервисах.
* Проект работает в трёх окружениях (dev/test/prod).
* Подготовлены спецификации OpenAPI и тесты (POSTMAN или unit-тесты).

### Минимальный набор тестов

* Регистрация, вход, доступ к защищённым маршрутам.
* Создание и чтение заказа.
* Получение списка заказов с пагинацией.
